3. Create `src/layouts/BaseLayout.astro`:
   - Accept `title: string` and optional `description: string` props via `interface Props`
   - In frontmatter: import `'../styles/global.css'` and `'@fontsource-variable/geist/wght.css'`
   - In frontmatter: import Header and Footer components
   - `<head>` section includes:
     - `<meta charset="utf-8" />`
     - `<meta name="viewport" content="width=device-width" />`
     - Dark mode FOUC prevention `<script is:inline>` (MUST be is:inline to run before paint):
       ```
       document.documentElement.classList.toggle(
         "dark",
         localStorage.theme === "dark" ||
           (!("theme" in localStorage) &&
             window.matchMedia("(prefers-color-scheme: dark)").matches)
       );
       ```
     - `<title>{title}</title>`
     - Conditional `<meta name="description">` if description prop provided
   - `<body>` with: `<Header />`, `<main>` with `<slot />`, `<Footer />`
   - Apply `font-sans` class to body for Geist font
   - Dark mode body styles: `bg-white dark:bg-neutral-950 text-neutral-900 dark:text-neutral-100` (or similar high-contrast pairing)
   - Responsive main content: max-width container with horizontal padding, centered

4. Add brutalist base styles to `src/styles/global.css` (append to existing content, do NOT remove the existing @import, @custom-variant, or @theme blocks):
   - Keep additions minimal — prefer Tailwind utility classes in components over custom CSS
   - If needed, add a few base overrides for the brutalist feel: e.g., aggressive heading sizes, tight body line-height

CRITICAL:
- The FOUC prevention script MUST use `is:inline` — without it, Astro bundles and defers the script, causing a visible flash
- The dark mode toggle script in Header.astro should NOT use `is:inline` — it can be bundled since it runs after the page loads
- Do NOT use `@variant dark` — it is `@custom-variant dark` (already in global.css from Plan 01)
